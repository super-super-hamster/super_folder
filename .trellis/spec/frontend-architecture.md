# Frontend Architecture

This document defines the core conventions for the React (TypeScript) frontend of the File Manager project.

## State Management

We use **Zustand** exclusively for global state management. Do not introduce Redux or Context API for state that changes frequently.

### Separation of Concerns
State is strictly separated by domain:
- `useUIStore` (`frontend/src/store/uiStore.ts`): Handles purely visual state (e.g., sidebar widths, search panel toggle, layout view modes).
- `useTabsStore` (`frontend/src/store/tabsStore.ts`): Handles domain logic for file browsing (e.g., active paths, history stacks, tab management).

**Pattern to Follow:**
Always extract state updates into the store actions rather than spreading `set()` logic across React components.

```typescript
// Good: Clear intent through store action
const { setSearchPanelOpen } = useUIStore()
setSearchPanelOpen(true)
```

## UI and Styling

We use **TailwindCSS** and **Framer Motion**.

### Avoiding the "AI Aesthetic"
Do not use generic UI patterns often generated by AI (e.g., oversized padding, excessive purple gradients, unnecessary shadows).

- **Strict Adherence to Custom UI Elements**: We have specific premium UI components, such as the *Search Panel Tag Capsules* and *Remark Panel*. 
- **Reference File (`TopNav.tsx`)**: Demonstrates how we parse the `searchQuery` and render inline, dark-gray semantic capsules (`tag:a`) seamlessly inside the input boundary.
- **Reference File (`RemarkPanel.tsx`)**: Demonstrates our preferred inline-editing aesthetic. Instead of standard save/cancel buttons with borders, we use clean transparent textareas (`bg-transparent text-sm text-gray-700 outline-none`) that autosave on `Blur` or `Enter`, and minimalistic rotating icon buttons (`motion.svg` `rotate: 45`) for toggling and deleting.

## Wails Integrations

Frontend components communicate with Go strictly via the auto-generated Wails bindings (`wailsjs/go/...`).

- Never mock these endpoints for production logic; use the real bindings.
- Handle all promises correctly. If an error is returned from a Go method (e.g., `ReadDir`), catch it and display a UI notification via `useModalStore.getState().openModal('warning', ...)` (as seen in `FileList.tsx`).
