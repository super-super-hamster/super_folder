# 批量重命名 (Batch Rename) 功能设计文档

## 核心目标
提供一个高效、直观且高度自定义的批量重命名页面。通过结合傻瓜式的基础输入框与安全的 JavaScript 执行引擎沙箱，既满足普通用户的常见需求，又能通过纯逻辑代码覆盖高级用户的复杂场景。

## UI 与交互设计
采用三列式设计：
1. **左侧 (原名列表)**：显示选中的原始文件名称。
2. **中间 (规则编辑区)**：
   - **基础模式 (可视化控制)**：提供常见的直观输入框，如“查找文本”、“替换为”、“加前缀”、“加后缀”、“起始序号”等。
   - **高级模式 (代码引擎)**：提供基于 `react-simple-code-editor` 和 `prismjs` 的 JS 代码框。
   - **关键联动机制**：基础模式和高级模式并非割裂。当用户在基础模式的输入框中打字时，下方的 JS 代码框会**实时自动生成对应的代码**。这不仅能无缝处理普通需求，还能作为高级代码的学习模板。
3. **右侧 (预览与检测)**：
   - 即时通过安全的隔离作用域计算新名称并渲染结果列表。
   - **冲突与错误处理**：
     - 如果生成的新名字数组中存在**重复名称**。
     - 如果生成的名字中包含**非法字符**（如 `\ / : * ? " < > |` 等）。
     - 以上两种情况都会使得对应行的背景色变为**红色警示**，并在界面上显示错误原因，同时“应用”按钮将被强制禁用。

## 技术实现细节
### 1. 状态流转
- **触发入口**：用户在 `FileList` 选中多项后按 `F2` 或点击右键菜单。
- **状态管理**：新增 `useRenameStore` 暂存选中文件，切换到 `batch-rename://` 标签。

### 2. 安全的 JS 引擎隔离
利用前端原生 V8 引擎解析代码，仅用于纯文本的推算。代码片段如：
```javascript
function rename(file, index) {
  // file.name (原名, 不含扩展名)
  // file.ext (扩展名, 如 .txt)
  // file.createTime (创建时间)
  // file.modifyTime (修改时间)
  return file.name + '_' + index + file.ext;
}
```
完全沙箱化，禁止访问 DOM 或底层的系统 API，绝对避免安全漏洞。

### 3. 后端 API 支持
- `app.go` 新增 `BatchRenameFiles(paths map[string]string) error` 方法，确保批量执行。
- `undo/undo.go` 支持 `OpBatchRename`，实现“一键 Ctrl+Z 全部撤回”的原子操作。
