# 文件管理器设计与交互规范 (Design Requirements)

本文档整理自项目初期的需求说明以及后续的所有优化迭代，作为前端 UI/UX 的基准设计规范。

## 1. 基础框架与风格
- **技术栈**：React + TypeScript + Wails，采用 TailwindCSS 进行原子化样式管理，Framer Motion 负责流畅的物理级动画。
- **窗口无边框**：完全弃用 Windows 原生的自带标题栏（去除默认的关闭、最大化、最小化按钮框架），采用纯自定义 UI 渲染。
- **拖拽控制**：灰色背景区域（非面板区域）及顶部导航栏均支持拖拽窗口 (`wails-draggable`)；各功能面板（如文件列表、侧边栏、搜索下拉面板）则禁止拖拽 (`wails-no-drag`)，防止误触。
- **状态记忆**：应用程序在关闭前会自动记录当前窗口的尺寸（宽高）和位置（X/Y 坐标），在下次启动时无缝恢复。
- **右键菜单与快捷键**：全局拦截并屏蔽浏览器默认的 Web 右键菜单，仅在特定区域允许自定义右键菜单。全面接管并禁用了浏览器原生快捷键（如 `F5`、`Ctrl+R`、`Ctrl+S` 等）以及 `Ctrl+滚轮` 页面缩放功能，以保障纯粹的桌面级原生软件体验。

## 2. 布局区域说明

### 2.1 左侧导航栏 (Left Sidebar)
- **行为**：默认收起，仅显示居中的图标；当鼠标指针移入（Hover）时，平滑展开并显示对应的文字标签。
- **内容**：包含本地磁盘入口（如 C盘、D盘，应用启动时默认进入 C 盘）以及桌面、图片、视频等系统默认路径。不再显示额外的“主文件夹”概念。
- **视觉**：收起状态下，由于内部文字通过 `width: 0` 彻底隐藏，图标能够获得完整的 24px 空间并绝对居中，无任何视觉挤压。

### 2.2 顶部功能栏 (Top Navigation)
从左至右依次为：
1. **历史控制**：包含后退、前进按钮，禁用时降低透明度。
2. **标签页 (Tabs)**：
   - 动态横向排布，展示当前所处的最深层目录名称（如 `a/b/c` 仅显示 `c`）。如果宽度超出，隐藏滚动条（`no-scrollbar`）并允许滚轮滑动查看。
   - 新建 Tab 的“加号”与关闭 Tab 的“叉号”默认隐藏，仅当指针移入时展示。
   - 具有轻快不粘腻的切换体验，切换 Tab 时拥有 Apple 风格的白底滑块动画（依托 `layoutId`）。
3. **搜索框**：
   - 默认处于收缩状态（仅显示图标和占位符）。
   - 只有在获取焦点时才会展开，展开时借助 `flex-1` 动态占据右侧**所有剩余宽度**，并且搜索框的底色保持灰底不变色。
   - 下方的搜索下拉面板（近期搜索）不再盲目展开，**只有**在获取焦点且输入了实质性搜索词时才会弹出。
   - **快捷键接管**：用户按下 `Ctrl+F` 时拦截系统查找框，并直接将焦点定位到该内部搜索框中。
4. **系统控制区**：
   - 包含排序/分组下拉菜单（按时间、名称排序，以及启用分组的勾选开关）。
   - 包含刷新按钮（点击拉取最新目录数据）、窗口控制按钮组（最小化、最大化、关闭）。
   - 该区域严格配置了防挤压逻辑（`shrink-0`），确保在搜索框获取焦点发生弹性伸缩时，所有的外围按钮均维持完美的圆形状态，绝不发生形变或扭曲。

### 2.3 主文件列表 (Main File List)
- **间距与尺寸**：列表容器左右及上方留有充裕的内边距（Padding）以增强呼吸感。采用响应式网格布局。
- **悬浮态 (Hover)**：指针移入文件图标时，视觉效果极为克制：仅显示灰色的圆角背景（`hover:bg-gray-100`），去除了任何夸张的放大缩放（Scale）和阴影（Shadow），尽显现代高级感。
- **框体约束**：每个文件热区卡片被严格限制了最大宽度（如 `w-28` 约 112px）并保持自身居中，防止在超大宽屏的网格下文件框被异常拉长。
- **滚动状态维持**：离开某文件夹后再次通过“后退”或 Tab 切换回来时，文件列表会自动定位还原到之前滑动到的具体垂直高度。
- **选择模式与复选框 (Selection Mode)**：
  - 复选框采用 HeroUI 复合组件构建。完全去除了默认阴影（`shadow-none`），外边框采用加粗轮廓（`border-2`）且被强制设置为正圆形（`rounded-full`），在尺寸上手动收紧（`w-[18px] h-[18px]`）以完美贴合内部的圆角勾号图标，选中时呈现明显的蓝色边框。
  - **防误触保护**：在多选模式下，点击文件列表的空白区域**不会**清空现有选中项或退出选择模式，保障了用户辛苦挑选的多文件状态不会因误点而丢失。

### 2.4 右侧功能栏 (Right Sidebar)
- **交互逻辑**：主要用于展示文件预览、信息或高级操作。由一个绝对定位在右侧边缘垂直居中的按钮（< / > 箭头）控制展开/收起。
- **位置固定**：该侧边栏开关按钮脱离了文件列表的内部滚动区域，固定悬浮于窗口绝对右侧，不会随着上下滚动而位移。
- **标签切换动画**：与其内部的“预览/信息/高级”三栏切换一样，享有丝滑的白色背景块物理滑动跟随动画。
